### Step 8: Identified Root Causes

**Primary Issue: In-Memory Rate Limiter**
- `lib/rate-limit.ts` uses `new Map()` for tracking attempts
- Vercel serverless functions are stateless
- Each cold start creates a new Map instance
- When accessed from different regions, Vercel may route to different edge instances
- Rate limit state is lost, causing inconsistent behavior

**Secondary Issue: No Environment Validation**
- `app/admin/(dashboard)/layout.tsx` calls `getSession()` without checking env vars first
- If env vars are missing in certain edge regions, Prisma/auth calls fail
- Error boundary catches the error but doesn't show which specific variable is missing

**Tertiary Issue: No Diagnostic Tools**
- No health check endpoint to verify environment and database status
- Difficult to debug what's failing in production

**After:**
```typescript
import { cookies } from 'next/headers';

export async function checkRateLimit(ip: string): Promise<{ allowed: boolean; remaining: number; resetInMs: number }> {
  const now = Date.now();
  
  try {
    const cookieStore = await cookies();
    const rateLimitCookie = cookieStore.get('rate_limit_' + ip.replace(/[^a-zA-Z0-9]/g, '_'));
    
    if (rateLimitCookie) {
      const data = JSON.parse(rateLimitCookie.value);
      if (now < data.resetAt) {
        if (data.count >= MAX_ATTEMPTS) {
          return { allowed: false, remaining: 0, resetInMs: data.resetAt - now };
        }
        data.count++;
        cookieStore.set('rate_limit_' + ip.replace(/[^a-zA-Z0-9]/g, '_'), JSON.stringify(data), {
          httpOnly: true,
          secure: process.env.NODE_ENV === 'production',
          sameSite: 'lax',
          path: '/',
          maxAge: Math.ceil((data.resetAt - now) / 1000),
        });
        return { allowed: true, remaining: MAX_ATTEMPTS - data.count, resetInMs: data.resetAt - now };
      }
    }
    
    const newData = { count: 1, resetAt: now + WINDOW_MS };
    cookieStore.set('rate_limit_' + ip.replace(/[^a-zA-Z0-9]/g, '_'), JSON.stringify(newData), {
      httpOnly: true,
      secure: process.env.NODE_ENV === 'production',
      sameSite: 'lax',
      path: '/',
      maxAge: Math.ceil(WINDOW_MS / 1000),
    });
    return { allowed: true, remaining: MAX_ATTEMPTS - 1, resetInMs: WINDOW_MS };
  } catch {
    return { allowed: true, remaining: MAX_ATTEMPTS, resetInMs: WINDOW_MS };
  }
}
```

**Why:** Cookies persist across serverless cold starts and edge regions. In-memory Map does not.

## Technical Analysis

### Why Timezone/Region Causes This Issue

1. **Vercel Edge Network:**
   - Vercel routes requests to the nearest edge location
   - Different regions may have different serverless instances
   - Each instance has its own memory space

2. **In-Memory State Problem:**
   - `new Map()` is stored in function memory
   - Serverless functions are stateless between invocations
   - Cold starts create fresh Map instances
   - Rate limit state is lost across regions

3. **Environment Variable Propagation:**
   - Vercel injects env vars at build time and runtime
   - In rare cases, edge functions may not have all vars loaded
   - This can happen during deployments or in specific regions

4. **Database Connection:**
   - Supabase PostgreSQL is in ap-south-1 (Mumbai)
   - Users in other regions may experience connection timeouts
   - pgBouncer connection pooling may behave differently per region

### Why Cookie-Based Rate Limiting Fixes It

1. **State Persistence:**
   - Cookies are stored in the user's browser
   - Persist across all edge locations
   - No dependency on server memory

2. **Cross-Region Consistency:**
   - Same cookie sent to all edge locations
   - Rate limit state follows the user
   - No state loss on cold starts

3. **Security:**
   - httpOnly flag prevents JavaScript access
   - secure flag ensures HTTPS-only in production
   - sameSite: 'lax' prevents CSRF attacks
