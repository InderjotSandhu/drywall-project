## Changes Made

### 1. Mobile Header Height Fix
- **Root cause:** On mobile, `.mobileMenu { display: flex }` was always in the DOM flow inside `<header>`. Even when visually hidden (translateY(-100%) + opacity: 0), it occupied layout space, making the header ~156px instead of the intended 56px.
- **Fix:** Added `position: absolute; left: 0; right: 0; top: 100%` to `.mobileMenu` (`Navbar.module.css`) — removes it from document flow so header height stays at `nav { height: 56px }` on mobile regardless of menu state.
